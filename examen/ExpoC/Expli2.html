L'histoire

Le langage C développé dans les années 70 par Ken Thompson et Dennis Ritchie est aujourd’hui l’un des langages de programmation les plus utilisés pour développer des programmes.

Le compilateur

Le compilateur est un traducteur, il fait le lien entre le langage C et le langage machine du processeur.

La compilation

Les différents types de fichiers utilisés lors de la compilation sont distingués par leur suffixe.

Les logiciels

Soit on installe séparément un éditeur de text, un compilateur et un debogueur.
Soit on installe un Environnement de développement.

La base d'une fonction

Il est nécessaire de taper sur la touche « Entrée » après la dernière accolade.
Chaque fichier en C devrait normalement se terminer par une ligne vide.
Si vous ne le faites pas, ce n'est pas grave, mais le compilateur risque de vous afficher un avertissement (warning).

Les différents types de variables existantes

Signed char, int et long permettent de stocker des nombres entiers
Float et double permettent de stocker des noms décimaux
unsigned permettent de stocker des nombres positifs uniquement

Fonction age

scanf demande une valeur à l'utilisateur, %d recupère la valeur indiqué par l'utilisateur

Fonction heures et minutes

& devant un nom de variable, on obtient son adresse au lieu de sa valeur.
* devant un nom de pointeur, on obtient la valeur de la variable stockée à l'adresse indiquée par le pointeur.

Fonction tableau

Les tableaux sont des ensembles de variables du même type stockées côte à côte en mémoire.
La taille d'un tableau doit être déterminée avant la compilation, elle ne peut pas dépendre d'une variable.

Mots clés a retenir

  • Code source : C'est le code de votre programme écrit dans un langage de haut niveau. C'est donc vous qui écrivez le code source, qui sera ensuite traduit en binaire.

  • Compilateur : C'est un programme qui permet de traduire le langage C en binaire. il existe un compilateur différent pour chaque langage de haut niveau.

  • Compilation : C'est la traduction du compilateur.

  • L'éxécutable : C'est le programme binaire créé par le compilateur, d'où l'extension .exe .

  • Debogueur : C'est un programme qui aide a traquer les erreurs.

  • Directives de préprocesseur : Ce sont des lignes spéciales que l'on ne voit qu'en haut des fichiers source. Ces lignes sont facilement reconnaissables car elles commencent par un dièse#. Ce sont des lignes qui seront lues par un programme appelé préprocesseur.

  • Préprocesseur : C'est un programme qui se lance au début de la compilation.
